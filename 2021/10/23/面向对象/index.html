面向对象

[TOC]

1. 基础理解

面向对象不是一种只针对于php的方法，而是一种运用于多种语言中的方法，将原本单向执行的面向过程变为将各个执行环节分配给具体的对象，将代码模块化，使代码的读、写、维护更符合自然人的思考模式，更加方便。

2. 类

是定义对象的最外层（同时是基础分支）用于包裹属性、方法、常量，具有对象们都需要的共性。

使用 class 类名{内部需要的属性、方法、常量} 进行定义。

如果类需要针对自身的数据操作，则使用范围解析操作符类名::访问对象;

self ：是一种在类的内部用来代替类名的关键字，其效果与$this类似，在类内部访问静态成员。（意义在于方便维护）

3.对象

是实际使用时实现功能的具体单元，将类中的内容进行使用，化抽象为具体的主体，这一过程被称为实例化，对象也被称为实例。

使用 $变量名=new 类名(); 的形式创建对象并赋给变量以便使用（可无限new出新对象）。

通过 $被赋的变量->类中的属性/方法; 来使用类中的内容（调用时不要加$）

$this :是类内部自带的对象，因此可以访问类内部所有的属性、方法而不受修饰限定符的限制，起连接桥梁的作用

4. 类成员

指类中的内容结构

(1)属性

是在类中创建的变量，也被称为成员变量

使用 public/protected/private $属性名="赋予的值" 来进行创建

public/protected/private 是访问修饰限定符，用于控制内容被访问的位置

public 共有，可以在任意的位置访问

protected 受保护，只能在类的内部访问，使用时需要$this起桥梁作用，本质是为了继承而出现的，可以被继承

private 私有，只能在”当前”类中访问，不可继承

(2) 方法

是在类中创建的函数，也被称为成员函数

使用public/protected/private function 方法名(){} 来进行创建

问修饰限定符的作用与属性类似

(3)类常量

是在类中创建的常量，由类访问，并不会在new对象时复制到对象的内存空间中（所以对象无法直接调用）

使用 public const 常量名=常量值; 来进行创建

如果要使用类常量则需要范围解析操作符 类名::常量名;

5.基本过程

  1. 将代码编译入内存(代码段)中
  2. 因为类是结构，不会执行，所以执行对象的相关操作
  3. new的对象前往内存(代码段)中寻找对应的类，把类中的所有属性复制到内存(堆区)中，并留下一个路径指针；被赋值的变量在内存(栈区)中有一路径指针指向内存(堆区)中的对象
  4. 当被赋值的变量使用类中的方法时，会沿着路径进入内存(代码段)中寻找方法，同时生成 $this

指向内存(堆区)中的对象，所以 $this 与被赋值的变量同时指向同一内存地址，代表同一对象，所以 $this 起到了桥梁的作用

6. 构造方法/析构方法

这两种方法属于PHP中的魔术方法（可以在满足条件时自动使用，输出的值随条件改变而改变）

属性需要初始化的情况是某个属性需要是统一的值，此时就需要构造方法__construct(){} 来进行初始化，此方法可以在new出新变量时就被执行

使用方法为：public function __construct($形式参数){$this->需要初始化的属性名=$形式参数;} new 类名(需要传入的值);

当需要消除某个对象并执行某个方法时，可以使用魔术方法__destruct(){} ，此魔术方法会在该对象被消除时自动执行（垂死挣扎)

7.静态成员

静态成员是针对类的成员，包含静态属性、静态方法。与类常量相似，不会在new对象时复制到对象的内存空间中（所以对象无法直接调用）

定义时在访问修饰限定符后加上static 即可

使用时直接由类来使用范围解析操作符类名::访问对象 访问对应的静态属性、静态方法。

由于不会占用新的内存空间，所以静态的效率更高。

在类的内部进行调用时，使用类名::访问对象;进行调用。

self ：是一种在类的内部用来代替类名的关键字，其效果与$this类似，在类内部访问静态成员。（意义在于方便维护）

8.类的加载

在正常的使用中，为了方便管理，会将一个类单独分入一个文件夹内，需要使用时再在对应的PHP文件中加载对应的类。有两种加载模式

手动加载：直接通过文件包含引入对应文件，其前可以使用if判定文件是否存在if(!class_exists(类名)){include_once'文件名'}

这种方法的缺点是过于繁琐且占用资源

自动加载：是指按规定好的路径去寻找对应的文件实现包含。

此功能原先由魔术方法__autoload 来实现，但随着版本更新，改为了使用函数spl_autoload_register(); 来进行注册设置好的加载函数。

9. 三大特性

1.封装

指类将其的成员包裹后在外部调用时不清楚运行的过程

2.继承

是指子类通过继承父类后可以调用父类的成员，包括静态成员（除私有private成员外）同时也可以增加新的成员

通过class 子类名 extends 父类名{} 实现继承，受保护的成员使用时同样需要$this的桥梁作用。

(1)重写

是指在子类中定义与父类同名的属性或方法，在new子类时，便会调用子类中重写的同名新成员。如果需要同时使用父类的同名方法使用关键字parent::父类的同名方法;进行调用（不能访问普通属性）PS：1.重写的成员的访问修饰限定不能高于父类。2.重写的方法必须与父类保持参数一致。

特点：php中只允许单继承（其他语言或许可以多继承），如有需要也只能采用套娃的形式（链式继承）

(2)静态延迟绑定

在静态成员在被继承时，就算重写了父类的静态属性，但在调用其静态方法时，由于父类中的self仍然是代表父类，所以输出的结果仍是父类的。但如果父类中使用静态延迟绑定的关键字static来代替self，则可以延迟关键字与类的绑定至在外部调用时，此时子类输出的结果便是子类重写后的。

(3)最终的final

final class 类名{}关键字的作用有两种，在类头中使该类无法被继承；在方法头中使该方法无法被重写。

(4)抽象abstract

abstract class 类名{}关键字会将修饰的类、方法抽象，抽象类无法实例化，只能被继承；抽象方法只能在抽象类中存在。在抽象类被继承时，子类必须实现父抽象类中的抽象方法（除非子类也是一个抽象类）。该关键字的意义是规范化类的继承，使子类实现统一的结构。ps：抽象方法后的括号一定要加分号。

(5)接口

接口与抽象相似，也是用来规定统一子类格式的一种结构，但接口不是类（自然不能new）。接口使用interface 接口名{接口常量;共有抽象方法;}（方法后的括号一定要加分号）来进行设置，限定类使用class 类名 implements 接口名{} 进行限定，接口中的方法都是抽象的。接口也可以被继承给其他的接口，并且接口可以多继承。

(6)trait

一种类似类的结构，但不能被实例化，是用来实现代码复用的，为其他的类提供公共的代码（方法），拥有属性与方法。创建为trait 名字{}，使用时在类中直接使用use 对应的名字;即可，相当于将trait中的方法直接复制进入当前类中，并且可以多引用 。ps：出现同名方法的情况时，优先级的顺序为本类的>trait>父类的，若是两个trait的方法重名，则在类中使用

回避同名。
